## TASK 2 — FREE_SESSIONS → 5 with monthly reset

**Files:**
- `src/lib/session-shared.ts` — change `FREE_SESSIONS = 3` → `5`.
- DB migration (Supabase MCP, `apply_migration`) — add columns + backfill.
- `src/lib/session-server.ts` or the session gate (wherever `if (used >= FREE_SESSIONS)` lives in `src/app/api/reid/route.ts:409-428`) — add monthly reset preamble.
- `src/app/(app)/chat/page.tsx`, `src/components/AppShell.tsx`, `src/app/(app)/settings/page.tsx` — update display to read `sessions_used_this_month`.

Migration:
```sql
ALTER TABLE public.users
  ADD COLUMN IF NOT EXISTS sessions_used_this_month integer NOT NULL DEFAULT 0,
  ADD COLUMN IF NOT EXISTS sessions_month_start timestamptz NOT NULL DEFAULT date_trunc('month', now() AT TIME ZONE 'UTC');
-- Backfill existing accounts so they start at 0 for the current month:
UPDATE public.users
  SET sessions_used_this_month = 0,
      sessions_month_start = date_trunc('month', now() AT TIME ZONE 'UTC')
  WHERE sessions_month_start IS NULL OR sessions_month_start < date_trunc('month', now() AT TIME ZONE 'UTC');
```

The gate in `src/app/api/reid/route.ts` runs the reset check first — if `now.getMonth() > monthStart.getMonth() || now.getFullYear() > monthStart.getFullYear()` then UPDATE row to 0 + first-of-month and treat the local count as 0. Then the existing `if (used >= FREE_SESSIONS) → 402` check uses `sessions_used_this_month` (NOT the lifetime `session_count` query). On a successful `createSession`, increment `sessions_used_this_month` by 1 via UPDATE (atomic SQL: `sessions_used_this_month = sessions_used_this_month + 1`). The lifetime `session_count` continues incrementing on SESSION_COMPLETE for analytics only. All three display surfaces read `me.sessions_used_this_month` and render `"Session {used + 1} of 5"` (chat header / sidebar) and `"{used} of 5 sessions used this month."` (Settings).
